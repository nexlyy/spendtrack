# Деплой SpendTrack как PWA (для iOS и не только)

Приложение полностью статическое (папка `web/static`) и работает офлайн (данные —
в IndexedDB на устройстве). Чтобы поставить его на iPhone без Mac, достаточно
выложить статику на любой бесплатный HTTPS-хостинг и «Добавить на экран Домой».

## Что выкладываем

Только папку **`web/static`**. В ней нет ни секретов, ни данных — чистый фронтенд.
Секреты (`.env`) и базы (`data/`) уже в `.gitignore`, в репозиторий не попадут.

> Расшаривать ссылку безопасно: данные у каждого устройства свои (IndexedDB),
> по ссылке отдаётся только само приложение, а не твои траты.

## Вариант A. GitHub → Vercel (рекомендую)

1. В корне проекта: `git init`, закоммить, запушь в репозиторий на GitHub.
   Репозиторий лучше **private** (Vercel деплоит и приватные бесплатно).
2. На <https://vercel.com> → **Add New → Project → Import** твой репозиторий.
3. Настройки импорта:
   - **Root Directory** → `web/static`  ← главное, иначе Vercel попробует собрать
     Python-бэкенд из `requirements.txt`.
   - **Framework Preset** → **Other**.
   - **Build Command** → пусто, **Output Directory** → пусто.
4. **Deploy** → получишь `https://твой-проект.vercel.app`.
5. Любой `git push` дальше — Vercel передеплоит автоматически. Пользователи
   получат обновление при следующем открытии.

## Вариант B. GitHub Pages (без Vercel)

1. Сделай отдельный репозиторий и положи в его корень **содержимое** `web/static`
   (чтобы `index.html` был в корне). Пути в приложении относительные — работает и
   из подпапки.
2. Settings → **Pages** → Source: ветка `main`, папка `/ (root)` → Save.
3. Через минуту будет `https://имя.github.io/репозиторий/`.

## Вариант C. Без git — Netlify Drop (самое быстрое)

<https://app.netlify.com/drop> → перетащи папку `web/static` → мгновенная
https-ссылка. Без аккаунта и без репозитория.

## Установка на iPhone

1. Открой https-ссылку именно в **Safari** (не в Chrome на iOS).
2. **Поделиться → На экран «Домой»**.
3. Запускай с иконки — полноэкранно и офлайн.

## Перенос данных между устройствами

Android/другой телефон → **⚙ → Сохранить копию** (файл `.json.gz`) → перешли себе →
на iPhone **⚙ → Восстановить из копии**.

## Нюанс iOS

iOS иногда чистит хранилище у редко используемых сайтов. У установленного на
«Домой» PWA данные живут стабильно, но привычка делать резервную копию лишней не
будет.
